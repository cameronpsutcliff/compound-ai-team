# Session-start benchmark

Generated by `measure.sh`. No metered API, no model call. Re-run
`bash measure.sh` to regenerate this file from the live source.

## Headline

Loading the full operating reference at session start costs about
**158.4x** the context this kit loads at session start. The kit reaches the
same capability by routing to one short procedure on demand instead of keeping
the whole reference resident.

| Set | Files | Bytes | Est. tokens |
|---|---:|---:|---:|
| Naive full reference (resident) | 135 | 808829 | 202207 |
| Kit session-start (tier-0 always-load) | 2 | 5105 | 1276 |
| **Ratio (naive / kit)** | | | **158.4x** |

Token figures are a character estimate (bytes / 4). They are
order-of-magnitude, not a tokenizer count; the ratio is what matters and it is
estimator-independent because both sets use the same divisor.

## What "naive" loads

The un-tiered baseline keeps the whole operating reference in context so every
capability is available without a router: the comprehensive field guide, every
skill procedure, every tier convention, every shell, and the root doctrine.

## What the kit loads at session start

Only the tier-0 always-load set:

- `AGENT.md`
- `doctrine/tiers/tier0.md`

The router then loads the one relevant procedure on demand. The reference is
present in the repo and reachable; it is simply not resident in every session.

## Method

1. `wc -c` over the two file sets listed above.
2. Token estimate = bytes / 4.
3. Ratio = naive bytes / kit bytes.

Pure shell. Reproducible on a bare laptop with no network and no API key.

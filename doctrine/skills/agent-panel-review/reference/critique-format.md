# agent-panel-review: Critique Format

The critique template is the discipline that keeps cross-critique
(Stage 3) from becoming negging. Every reviewer covers four cells per
output reviewed. No free-form paragraphs of complaint.

---

## The four-cell template

For each output reviewed:

```
CRITIQUE OF [Panelist X]'s OUTPUT

Strongest claim:
  [The single most defensible move in this output.
   Specific. Cite the line or section.]

Weakest claim:
  [The single most vulnerable move in this output.
   Specific. Name the vulnerability, not a vague worry.]

Shared blind spot:
  [Something this output and yours both missed.
   This is the most valuable cell. It surfaces what no one saw.]

One thing worth stealing:
  [What you would adopt from this output into yours.
   Concrete. A framing, a phrase, a structural move.]
```

Each cell must be specific. "The intro is weak" fails. "The intro
opens on a definition instead of an incident, which loses the reader
before the thesis lands" passes.

---

## Why this format works

**One cell forces strength-recognition.** Without the strongest-claim
cell, critique becomes a list of complaints. With it, every reviewer
has to find a real strength, which calibrates the negative critique
that follows.

**One cell forces specific weakness.** Without the weakest-claim cell,
critique stays at the safe "this could be improved" level. With it,
the reviewer has to name the single weakest point, which means the
critique is actionable.

**The shared-blind-spot cell is where the panel earns its cost.** This
is the cell that an individual agent could not produce. It requires the
reviewer to read someone else's output through the lens of what their
own output also missed. The blind-spot surfaces are what convergence is
made of.

**The one-thing-to-steal cell prevents convergence on the average.**
Without it, the panel might converge by everyone keeping their own
draft and ignoring the others. With it, every revision in Stage 4
incorporates a specific concrete move from another panelist's output.

---

## Anti-patterns in critique

### Anti-pattern: "this is fine"

Symptom: weakest-claim cell says "no major issues." Fix: every output
has a weakest claim. Find it. If genuinely no major issues exist, the
output is too generic and that itself is the weakest claim.

### Anti-pattern: critiquing the agent

Symptom: critique references the producing agent ("Codex's output is
weak on attribution"). Fix: critique the work. "This output is weak on
attribution."

### Anti-pattern: stealing nothing

Symptom: every "one thing worth stealing" cell is hand-wavy. Fix: the
cell forces specificity. If you cannot name something concrete to
steal, you have not read the output closely enough.

### Anti-pattern: novel-length critique

Symptom: each critique cell is three paragraphs. Fix: the cell is one
to four sentences. Critique is signal, not essay.

### Anti-pattern: critique that re-proposes the output

Symptom: weakest-claim cell turns into "and here is how I would rewrite
it" with a 200-word rewrite. Fix: critique surfaces the weakness; the
producing agent revises in Stage 4. The reviewer does not rewrite.

---

## When to use scorecard instead

The four-cell template is the default at Stage 3. For heavier
deliverables where quantitative comparison across drafts justifies more
structure, swap in the **scorecard rubric** from
`reference/scorecard-rubric.md`: ten dimensions, 0-100 score per
dimension with evidence citation, composite + dual grade. Pick one
format per loop; do not mix within a single loop.

---

## Self-critique (optional but recommended)

After producing critiques of the other panelists, each agent can
produce a self-critique using the same template, applied to its own
output as if reading it fresh.

Self-critique is optional because the cross-critique loop already
forces revision. But agents that produce a self-critique tend to write
better Stage 4 revisions because they have already named their own
weakest claim out loud.

---

## What the operator does with critiques

The operator reads all critiques before sending them to the panelists
for Stage 4 self-revise. The operator's job at this point:

1. Verify every critique uses the four-cell template (reject any that
   do not, ask for redo)
2. Check for "critiquing the agent" anti-pattern; reject if found
3. Bundle each panelist's incoming critiques (from the other two
   panelists) and send

The operator does not edit critiques. The operator gatekeeps format.

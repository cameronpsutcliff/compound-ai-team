# Transform rules for derive.sh

Reproducible scrub transforms applied when deriving Individual and Team editions
from the Mini canonical. Downstream trees are generated by `derive/derive.sh`,
never hand-edited. If a transform changes, update this file and re-run derive.

Authority: `attribution-allowlist.txt` and `leak-denylist.txt` on the canonical.
The derive step applies **denylist-minus-allowlist**: strip allowlisted tokens from
each line, then drop any line that still matches a denylist term or structural
personal-path pattern. This matches `enforcement/bin/check-portability.sh`.

## 3. Denylist-minus-allowlist scrub

After version and token rewrites, for each text file:

1. Load terms from `attribution-allowlist.txt` (longest first).
2. Load terms from `leak-denylist.txt` / `forbidden-terms.txt`.
3. For each line: remove every allowlisted substring, then discard the line if it
   still contains any denylist term or personal filesystem path.

Allowlisted co-author identities **must survive**: `Joshua Sutcliff`,
`joshuadsutcliff`, `Cameron Sutcliff`, `cameronpsutcliff`, and the public System B
reference `github.com/brass458/claude-config-public`.

## 5. Manifest regeneration

After copy and scrub, derive regenerates:

- `compound-ai.manifest.json` (file list, per-file sha256, aggregate hash, version)
- `compound-ai.sha256` (plain-text checksum listing)

Implementation: seed manifest metadata, then run
`reference-impl/scripts/build-manifest.py` in the derived tree.

1. `enforcement/bin/check-kit.sh` on the derived tree (all gates PASS).
2. `enforcement/bin/simulate-fresh-clone.sh <derived-tree>` (fresh-copy simulation).
3. `enforcement/bin/check-portability.sh` confirms allowlisted names present and denylist absent.
